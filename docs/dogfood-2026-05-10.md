## Bugs surfaced by the dogfood

The dogfood found one real install bug worth recording — exactly the
kind of finding the dogfood phase exists to surface.

**`install.sh` in symlink mode creates a sub-symlink inside the
source repo.** The script's "templates symlink" step writes
`$OPENCODE_CONFIG_DIR/skills/setup-opencode-agenticapps-workflow/templates`,
which (in symlink-install mode) resolves through the main symlink
back to `repo/skills/setup-opencode-agenticapps-workflow/templates`.
That side-effect appears as an untracked symlink in the source
repo's `git status`.

For Phase 6, mitigated via `.gitignore` rule
`skills/*/templates` so the side-effect doesn't pollute commits.
The symlink continues to be regenerated by `install.sh` and the
migrations resolve through it correctly.

**v0.2.0 follow-up:** restructure so `install.sh` in symlink mode
does not need to write inside the source tree. Likely path: move
the templates into `skills/setup-opencode-agenticapps-workflow/templates/`
permanently (committing them under that path in the repo), and drop
the secondary symlink step. Migrations then read from a stable path
without install-time writes.
